# คู่มือการอัพเดทไฟล์ขึ้น GitHub ฉบับสมบูรณ์ (สำหรับอนาคต)

หากคุณ Beam ต้องการอัพเดทงานขึ้น GitHub ในครั้งต่อไป แนะนำให้รันคำสั่งตามลำดับด้านล่างนี้ครับ:

### 1. เข้าไปยังโฟลเดอร์งาน

เปิด Terminal แล้วเข้าไปที่โฟลเดอร์โปรเจกต์ก่อนเสมอ:

```powershell
cd "c:\Users\USER\OneDrive\Documents\BEAM\ANN\COE64-345 Big Data Implementation\Book Recommendation System"
```

### 2. ตรวจสอบข้อมูลประจำตัว (Identity)

เพื่อให้ GitHub รู้ว่าเป็นฝีมือของคุณ Beam (ทำซ้ำได้ถ้าเปลี่ยนเครื่องหรือโฟลเดอร์):

```powershell
git config user.email "apinan.ar@mail.wu.ac.th"
git config user.name "Apinan1234"
```

### 3. เพิ่มไฟล์และบันทึกงานใหม่ (Add & Commit)

เพื่อเตรียมไฟล์และใส่โน้ตว่าอัปเดตอะไรไป:

```powershell
# เพิ่มไฟล์ทั้งหมดในโฟลเดอร์
git add .

# บันทึกงานพร้อมข้อความอธิบาย
git commit -m "อัปเดตเนื้อหาโปรเจกต์ Book Recommendation System"
```

### 4. ส่งไฟล์ขึ้น GitHub (Push)

เพื่อส่งงานจากเครื่องขึ้นไปที่ GitHub ของจริง:

```powershell
# หากเป็นการอัปเดตปกติ
git push

# หรือหากต้องการบังคับให้ GitHub เป็นเวอร์ชันเดียวกับในเครื่อง (ใช้กรณีติด Error)
git push -u origin main --force
```

---

### ⚠️ ข้อควรระวังเพิ่มเติม

1. **ไฟล์ขนาดใหญ่:** ห้ามใส่ไฟล์ที่ใหญ่เกิน **100MB** (เช่น วิดีโอ หรือ Dataset ขนาดใหญ่มาก) เพราะ GitHub จะบล็อกการอัปโหลด
2. **การลบไฟล์:** หากมีการลบไฟล์ในเครื่อง Git จะจำสถานะการลบไปแสดงบน GitHub ด้วยหลังจากรัน `git add .` และ `push`

**เก็บคู่มือนี้ไว้ใช้สำหรับโปรเจกต์ต่อๆ ไปได้เลยครับ!**
