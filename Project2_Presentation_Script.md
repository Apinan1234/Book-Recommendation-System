### [ Slide 2: Data & อธิบาย Data ]

**ผู้พูด:** *(เปิดสไลด์ Data)*
"เรามาเข้าสู่ส่วนแรกกันเลยครับ/ค่ะ นั่นคือชุดข้อมูลที่เราเลือกนำมาใช้
ในโปรเจกต์นี้ผม/ดิฉันได้ใช้ชุดข้อมูล **Goodbooks-10k** ซึ่งประกอบไปด้วยหนังสือยอดนิยมกว่า 10,000 เล่ม และมีประวัติการให้คะแนนรีวิว หรือ Rating จากผู้ใช้อีกมากกว่า 6 ล้านรายการเลยทีเดียวครับ

สาเหตุที่เลือกชุดข้อมูลนี้ เพราะว่ามันมีความเหมาะสมอย่างมากที่จะนำมาสร้างระบบแนะนำหนังสือ เพื่อทำนายว่าผู้ใช้งานลักษณะแบบนี้ น่าจะชื่นชอบหนังสือเล่มไหนตามมา โดยเราอ้างอิงและประมวลผลจากการให้คะแนนในอดีต (Collaborative Filtering) ครับ/ค่ะ"

### [ Slide 3: Field Description ]

**ผู้พูด:** *(เปิดสไลด์โครงสร้าง Data)*
"สำหรับส่วนของ Field Description นะครับ/คะ ชุดข้อมูลนี้จะถูกดึงมาใช้หลักๆ 2 ตารางด้วยกันครับ:

1. **ตารางแรก `ratings.csv`:** เป็นข้อมูลพฤติกรรมการให้คะแนน จะมีคอลัมน์สำคัญคือ `user_id` (ใครเป็นคนให้), `book_id` (ให้หนังสือเล่มไหน) และ `rating` (ให้คะแนนกี่ดาว 1-5 ดาวครับ)
2. **ตารางที่สอง `books.csv`:** คือตารางข้อมูลรายละเอียดหนังสือ ประกอบด้วย `book_id` เพื่อเอาไปจับคู่กับตารางแรก แล้วก็มีคอลัมน์ `title` ชื่อหนังสือ, `authors` ชื่อผู้แต่ง และความนิยมโดยรวมต่างๆ (เช่น `average_rating`, `ratings_count`) ครับ/ค่ะ"

### [ Slide 4: การวิเคราะห์ด้วย ]

**ผู้พูด:** *(เปิดสไลด์เครื่องมือที่ใช้)*
"แล้วเราใช้อะไรในการจัดการข้อมูลที่มีปริมาณมากระดับ 6 ล้านรายการแบบลื่นไหลล่ะ?

คำตอบคือ เราใช้ Framework อย่าง **PySpark (Apache Spark)** ในการทำ Distributed Data Processing เพื่อประสิทธิภาพความเร็วครับ/ค่ะ และตัวหัวใจของระบบ ผม/ดิฉันได้เลือกใช้อัลกอริทึม **Alternating Least Squares (หรือ ALS)** ภายใต้คอนเซปต์ของ **Collaborative Filtering** ในการสร้างโมเดล Machine Learning เพื่อหารูปแบบความคล้ายคลึงของรสนิยมคนอ่านครับ"

---

### [ Slide 5 - 6: ขั้นตอนการวิเคราะห์ ]

**ผู้พูด:** *(เปิดสไลด์ขั้นตอน)*
"สำหรับขั้นตอนการทำงานของโปรแกรมเราแบ่งออกเป็น 7 Steps ด้วยกันครับ:

1. เริ่มตั้งแต่ **Setup:** เราสร้าง SparkSession เพื่อจัดการการสร้าง Cluster เสมือน
2. **Data Loading:** นำเข้าตาราง CSV โหลดเป็นแบบ PySpark DataFrame
3. **Data Cleaning:** ลบข้อมูลพวก Null ที่อาจทำให้โปรแกรมพังในหลักสูตรออกครับ `user_id`, `book_id`, `rating`
4. **Train/Test Split:** อันนี้เป็นหัวใจเลยครับ เราสุ่มแบ่งข้อมูลออกเป็น Train 80% เอาไว้สร้างความรู้ให้โมเดล และดึง Test 20% แยกเก็บไว้แอบสอบวัดผลตอนจบครับ
5. **Model Building:** สั่งรัน ALS กำหนด 10 รอบ (maxIter) เพื่อหากลุ่มผู้ใช้ที่มีรสนิยมสไตล์การอ่านตรงกัน
6. **Evaluation:** เอา Test 20% มาวัดค่า RMSE ว่าโมเดลแม่นไหม
7. และสุดท้าย **Generate Recommendations:** สั่งทำนาย Top 5 แนะนำหนังสือให้กับผู้ใช้แบบรายบุคคลเลยครับ"

### [ Slide 7: อธิบายโค้ดที่รัน ]

**ผู้พูด:** *(ชี้หรือโชว์ภาพ Code ย่อๆ)*
"ในส่วนของ Source Code เพื่อให้เห็นภาพชัดเจน ผม/ดิฉันขอหยิบยกคำสั่งหลักๆ มาอธิบายครับ

โค้ดบรรทัดหลักของเราจะใช้คำสั่งของ PySpark ตั้งแต่ `spark.read.csv` ในการโหลด Data เข้าไป
ส่วนที่สำคัญที่สุดคือบรรทัด `ALS(maxIter=10, regParam=0.1, userCol="user_id", itemCol="book_id", ratingCol="rating")` ตรงนี้เป็นการประกาศเรียกใช้ตัวอัลกอริทึมในการทำ Recommendation ครับ โดยกำหนด Column เป้าหมาย จากนั้นเราใช้ `als.fit(trainingData)` เพื่อให้มันนั่งเรียนรู้ความสัมพันธ์ของหนังสือกับนักอ่านกว่าหลายล้านเรคคอร์ด

หลังจากเทรนโมเดลเสร็จ เราจึงสั่ง `evaluator.evaluate(predictions)` เพื่อวัดค่าความแม่นยำ และใช้ `model.recommendForUserSubset` ในการโชว์ผลลัพธ์ Top 5 Recommendation ปิดท้ายเพื่อให้มันพ่นรายชื่อหนังสือแนะนำ 5 อันดับแรกของผู้ใช้แต่ละคนออกมาครับ"

### [ Slide 8: ผลลัพธ์ที่ได้ และอธิบายทีละขั้นตอน ]

**ผู้พูด:** *(เปิดสไลด์รูป Terminal Screenshot หรืออ่านไปตามภาพ Output)*
"คราวนี้เรามาดูหน้าตากระบวนการรันผลลัพธ์ออกมาใช้งานจริงใน Terminal ตามทีละสเตปกันเลยครับ

- **STEP 1-2:** ตัวโปรแกรมจะโชว์ว่า เราเริ่ม Spark Session แล้วนะ แล้วดึงพรีวิวตาราง User + Book + Rating 3 บรรทัดแรกออกมาให้ดูว่าไฟล์โหลดติดจริงๆ
- **STEP 3-4:** ส่วนตัวระบบจะแอบไปจับข้อมูลแยก Train 80/20 ให้ แล้วโปรแกรมจะทำการขมวดเอาข้อมูลเข้า ALS Train Model อย่างตั้งใจ
- **STEP 5:** ตรงนี้คือทีเด็ดครับ ได้ผลลัพธ์คะแนนสอบโมเดลหน้า Terminal เราได้ค่าการประเมินยอดฮิตอย่าง **RMSE ออกมาที่ตัวเลข 0.8195**
(ถ้าอาจารย์ถามว่า 0.8195 คืออะไร?) ตอบได้เลยว่า 'คือค่าความคลาดเคลื่อนเฉลี่ยของการทำนายดาว' ข้อมูลของเราคะแนนเต็ม 5 ดาว โมเดลเดาดาวผิดปกติแค่บวกลบไม่เกิน 1 ดาวเลย ซึ่งเป็นโมเดลที่แม่นและเริ่มใช้งานได้ดีเลยครับ
- **STEP 6:** เป็น Output ชั้นยอดที่เรารอคอยเลยครับ โปรแกรมจัดตารางให้เรียบร้อย โดยดึง User Number สุ่มมา 5 คน (เช่น ID: 463, 1238, 1580) แล้วทำนายดาวที่พวกเค้าน่าจะโหวตสูงสุดมาจัดอันดับ 5 ผลลัพธ์ เช่น User เบอร์ 1238 ระบบเดาว่าจะต้องชอบหนังสือ "Kindle Paperwhite User's Guide" อย่างแน่นอน พร้อมโชว์คะแนน Predicted Rating ออกมาประกบครับ!"
