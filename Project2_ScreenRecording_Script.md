### [ 1. แนะนำตัว & อธิบาย Data ]

**ผู้พูด:** *(เปิดหน้าจอคอมพิวเตอร์ โชว์ไฟล์รายงาน หรือ รันโค้ดทิ้งไว้)*
"สวัสดีครับ ชื่อ [ชื่อ-นามสกุล] รหัสประจำตัว [รหัสประจำตัว] ครับ
วันนี้ผมจะมาอธิบายโปรเจกต์ที่ 2 ของวิชา Big Data Implementation ในหัวข้อ **Book Recommendation System** หรือระบบแนะนำหนังสือครับ

สำหรับโปรเจกต์นี้ ผมเลือกใช้ Data ที่ชื่อว่า **Goodbooks-10k** ซึ่งเป็นชุดข้อมูลที่รวบรวมข้อมูลหนังสือยอดฮิต 10,000 เล่ม และประวัติการรีวิวการให้คะแนนของผู้อ่านที่มากกว่า 6 ล้านรายการครับ เหตุผลที่เลือก Data ชุดนี้มาเพราะมันตอบโจทย์การทำระบบ Collaborative Filtering มากๆ คือเราจะให้โปรแกรมมันเดาใจคนอ่านว่า ถ้าเขาชอบหนังสือเล่มนี้ เขาควรจะโดนป้ายยาหนังสือเล่มไหนตามมาครับ"

### [ 2. Field Description ]

**ผู้พูด:** *(ชี้เมาส์ไปที่ส่วนคำอธิบาย Data ในรายงาน หรือ เปิดไฟล์ CSV ให้ดูแวบๆ)*
"ในชุดข้อมูลนี้ หลักๆ ผมจะดึงมา 2 ตารางครับ

- **ตารางแรก** คือ `ratings.csv` ตารางตัวนี้สำคัญสุดเลย เพราะมันเก็บ `user_id` ว่าใครเป็นคนอ่าน, `book_id` เค้าอ่านเล่มไหนไป, และ `rating` เค้าให้คะแนนความชอบหนังสือเล่มนั้นกี่ดาว (1 ถึง 5 ดาว) ครับ
- **ตารางที่สอง** คือ `books.csv` เอาไว้แค่ดึง `title` (ชื่อหนังสือ) กับ `authors` (ชื่อคนแต่ง) มารวมร่างโชว์ผลลัพธ์ตอนจบให้เราอ่านรู้เรื่องครับ"

---

### [ 3. การวิเคราะห์ด้วย & ขั้นตอน ]

**ผู้พูด:** *(เริ่มสลับแท็บไปที่หน้าไฟล์โค้ด `main.py`)*
"สำหรับเครื่องมือที่ใช้ เนื่องจากข้อมูลเรามันระดับ 6 ล้านเรคคอร์ด ผมเลยเลือกเอา **PySpark** ซึ่งเป็นเครื่องมือจัดการ Big Data เข้ามาช่วยประมวลผลครับ และอัลกอริทึมพระเอกของเราในงานนี้คือ **ALS (Alternating Least Squares)** ซึ่งจะเป็นตัวหาความสัมพันธ์ระหว่างคนอ่านกับหนังสือครับ

**ขั้นตอนการทำงานของโปรแกรม หลักๆ จะมี 7 สเตป ดังนี้ครับ:**

1. ติดตั้งและสร้าง SparkSession
2. Data Loading ไฟล์ CSV ทั้ง 2 ตัว
3. Data Cleaning กรองพวกค่าว่าง (null) ออก
4. แบ่งข้อมูล (Train/Test Split) สำหรับเทรนและทดสอบ
5. สร้างตัวโมเดล ALS
6. วัดผลประเมินความแม่นยำ
7. โชว์ผลลัพธ์การแนะนำหนังสือของแต่ละคนออกมาครับ"

---

### [ 4. อธิบาย Code ]

**ผู้พูด:** *(ชี้เมาส์ไฮไลท์บรรทัดของโค้ดที่อธิบายไปพร้อมๆ กันใน `main.py`)*
"เดี๋ยวผมจะรันทีละส่วนให้ดู โค้ดหลักๆ ของเราจะเริ่มบรรทัดนี้ครับ `spark.read.csv` เพื่อทำการดึงข้อมูลเข้าสู่ระบบ
จากนั้นผมจับมารวมร่างและลบค่าว่างออก แล้วเรามาสุ่มแยกข้อมูลตรงนี้ครับ `ratings.randomSplit([0.8, 0.2])` คือการจับแยกข้อมูล Train เอาไปสอนโมเดล 80% และเก็บเอาไว้ 20% ไว้สำหรับเป็นชุดข้อสอบประเมินโมเดลตอนจบว่าแม่นไหมครับ

และจุดพีคของโค้ดอยู่ตรงนี้ครับ โมเดลรันคำสั่ง `ALS(maxIter=10, regParam=0.1 ...)` เข้าใส่ Training Data
ให้มันเทรนเป็นเวลา 10 รอบ เพื่อหาความคล้ายคลึงของคนอ่าน ก่อนจะโยนคะแนน 20% ที่เก็บไว้มาเข้าฟังก์ชัน `evaluator.evaluate(...)` เพื่อวัดค่าคลาดเคลื่อนครับ"

---

### [ 5. ผลลัพธ์ที่ได้ และ อธิบายผลลัพธ์แต่ละขั้นตอน ]

**ผู้พูด:** *(สลับหรือชี้เมาส์ไปที่ช่อง Terminal ด้านล่างที่รันเสร็จเรียบร้อยแล้ว)*
"คราวนี้เรามาดูหน้าจอผลลัพธ์ของจริงบน Terminal ที่เพิ่งรันเสร็จกันเลยครับ ว่ามันพ่นอะไรออกมาบ้าง

- **สเตปโหลดข้อมูลเซต [1]-[2] :** โปรแกรมก็ลากข้อมูลมาแสดงดิบๆ ยืนยันว่าดึงไฟล์สำเร็จครับ
- **มันก็ข้ามมาสเตปที่ [5] ประเมินผลเลย:** ค่าที่ได้ตรงนี้เรียกว่า **RMSE** ของโปรเจกต์นี้ รันออกมาได้ **0.8195**
(RMSE คืออะไร?) อธิบายง่ายๆ คือค่าความคลาดเคลื่อนเฉลี่ยของการเดาดาวของโมเดลเราครับ เราให้คะแนนเต็ม 5 ดาว แปลว่าโมเดลเราเดาดาวคลาดเคลื่อนจากความเป็นจริงเฉลี่ยไม่ถึง 1 ดาวเลย ซึ่งถือว่าความแม่นยำในเกณฑ์ที่ยอดเยี่ยมมากครับสำหรับการทายใจมนุษย์
- **และสเตปที่ [6] อันนี้สวยงามมากครับ:** Output จะออกมาเป็นตารางรายชื่อหนังสือ 5 อันดับแรก (Top 5) ที่โมเดลจัดเตรียมมาประเคนให้ผู้ใช้งาน (User ID) แต่ละคนแบบไม่ซ้ำกันตามรสนิยมครับ ยกตัวอย่าง User ID: 1238 ระบบก็จะวิเคราะห์และเดาว่า ถ้า User คนนี้มาซื้อหนังสือ เขาตัองอยากอ่านเล่มแรกคือ 'Kindle Paperwhite' แน่นอน พร้อมโชว์คะแนนที่โมเดลเดาว่าเค้าจะชอบไว้ที่ 4.6 ดาวทางด้านขวาสุดด้วยครับ"

"จากผลลัพธ์ทั้งหมด **สรุป** ได้ว่าโปรเจกต์รายวิชา Big Data ของเราในครั้งนี้ ได้พิสูจน์ให้เห็นถึงประสิทธิภาพของการสร้าง Recommendation System ครับ การที่เราเอา PySpark เข้ามาจับคู่ประยุกต์ร่วมกับอัลกอริทึม Machine Learning อย่าง ALS
